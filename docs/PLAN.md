# PLAN.md

## 計畫目標

建立一個互動的網頁，包含輸入框、按鈕與影片顯示的功能。使用者可以輸入歌名，按下按鈕後，對應的影片卡片會顯示在頁面上。

---

## 實現步驟

### 1. 準備基本結構

- 在頁面上放三個主要部件：
  1. 一個輸入框，用於讓使用者輸入文字（例如：影片名稱或描述）。
  2. 一個按鈕，當被按下時會啟動動作。
  3. 一個空白的影片框準備顯示結果。
- **HTML 示意：**
  ```html
  <div>
    <input type="text" placeholder="請輸入歌名" />
    <button>搜尋</button>
    <div>
      <img src="" alt="這裡會顯示影片縮圖" />
    </div>
  </div>
  ```

---

### 2. 實現輸入框變化監測

- 輸入框是使用者的小「盒子」，儲存歌名。
- 當使用者在輸入框裡打字時，程式需要即時記住這些字。

**要點：**

- 用 Vue 綁定輸入框內容到程式的變數（用「偷看」的方式保存文字內容）。
- 提示：這一步讓輸入數據可以在全程使用。

---

### 3. 設定按鈕的行為

- 按鈕是「觸發開關」，點擊後開始搜尋。
- 按下時：
  1. 程式會讀取使用者輸入的歌名。
  2. 程式會把歌名送去搜尋，找出播放量最高的影片。

**步驟：**

1. 在 HTML 中綁定按鈕點擊事件。
2. 設定一個函數，讀取輸入框的歌名，並送去搜尋影片資料。
3. 確保傳遞錯誤時，顯示「查無此歌」或錯誤提示。

---

### 4. 圖片綁定與顯示

- 在程式中，將影片資料與輸入框的內容動態關聯起來。
- 使用者輸入歌名後，畫面會顯示影片縮圖、名稱、頻道、發布時間與觀看次數。
- 預設空值時，不顯示結果卡片。

**要點：**

- 將 `<img>` 元素的屬性 (src) 與影片縮圖綁定。
- 處理錯誤：如果找不到結果，顯示短句「查無此歌」或錯誤提示。

---

### 5. 測試與調整

- 檢查交互流程是否正確：
  1. 測試空白輸入時的行為。
  2. 測試正確輸入是否顯示對應影片卡片。
  3. 模擬錯誤場景，觀察是否出現提示。
- 修復潛在問題，例如按下按鈕沒反應或結果未顯示。
