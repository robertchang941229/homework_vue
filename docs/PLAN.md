# PLAN.md

## 計畫目標

建立一個互動的網頁，包含輸入框、按鈕與圖片顯示的功能。用戶可以輸入文字，按下按鈕後，指定的照片會顯示在頁面上。

---

## 實現步驟

### 1. 準備基本結構

- 在頁面上放三個主要部件：
  1. 一個輸入框，用於讓使用者輸入文字（例如：照片名稱或描述）。
  2. 一個按鈕，當被按下時會啟動動作。
  3. 一個空白的圖片框準備顯示照片。
- **HTML 示意：**
  ```html
  <div>
    <input type="text" placeholder="請輸入照片名稱" />
    <button>顯示照片</button>
    <div>
      <img src="" alt="這裡會顯示照片" />
    </div>
  </div>
  ```

---

### 2. 實現輸入框變化監測

- 輸入框是使用者的小「盒子」，儲存文字信息。
- 當用戶在輸入框裡打字時，程式需要即時跟蹤這些字並保存。

**要點：**

- 用 Vue 綁定輸入框內容到程式的變數（用「偷看」的方式保存文字內容）。
- 提示：這一步讓輸入數據可以在全程使用。

---

### 3. 設定按鈕的行為

- 按鈕是「觸發開關」，點擊後執行指定操作。
- 按下時：
  1. 程式會讀取使用者輸入的內容。
  2. 使用內容去更新圖片的「圖片來源路徑」。

**步驟：**

1. 在 HTML 中綁定按鈕點擊事件。
2. 設定一個函數，讀取輸入框的值，並更新圖片展示路徑。
3. 確保傳遞錯誤時，顯示「找不到圖片」的提示。

---

### 4. 圖片綁定與顯示

- 在程式中，將圖片的路徑與輸入框的內容動態關聯起來。
- 用戶可以輸入正確的文件名稱或者 URL，照片將自動展示。
- 預設空值，用佔位圖替代無效的內容。

**要點：**

- 將 `<img>` 元素的屬性 (src) 與輸入綁定。
- 處理錯誤：如果找不到照片，顯示短句「未找到圖片」。

---

### 5. 測試與調整

- 檢查交互流程是否正確：
  1. 測試空白輸入時的行為。
  2. 測試正確輸入是否顯示對應圖片。
  3. 模擬錯誤場景，觀察是否出現提示。
- 修復潛在問題，例如按下按鈕沒反應或圖片未顯示。
